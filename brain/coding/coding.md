## My Problem Solving Approach

Whenever I encounter a new problem, I usually follow a structured approach:

1. Understand the problem completely.
2. Think about the brute-force solution.
3. Identify the bottleneck.
4. Look for patterns or observations.
5. Optimize the solution.
6. Analyze time and space complexity.
7. Test edge cases before considering the solution complete.

I believe understanding *why* an algorithm works is much more valuable than memorizing it.

## Debugging Process

Debugging is an important part of software development.

When something doesn't work, I avoid making random changes. Instead, I debug systematically.

My usual debugging process is:

- Read the error message carefully.
- Reproduce the issue consistently.
- Isolate the problematic component.
- Check logs and intermediate outputs.
- Verify assumptions.
- Read documentation when necessary.
- Fix the root cause rather than only the symptoms.

This approach has helped me solve many backend, deployment, and API integration issues.

## Development Workflow

When building a project, I generally follow this workflow:

1. Understand the problem.
2. Plan the architecture.
3. Build a simple working version.
4. Test each component individually.
5. Improve the implementation.
6. Refactor the code.
7. Deploy the application.
8. Continue improving based on feedback.

I prefer iterative development over trying to build everything perfectly in one attempt.
